Quick Start with Maven

Add the SCell Public API and SCell API-impl dependencies to the POM:

    <dependencies>
        ...
        <dependency>
            <groupId>com.intechcore.scomponents.scell</groupId>
            <artifactId>api</artifactId>
            <version>0.1.1</version>
        </dependency>
        <dependency>
            <groupId>com.intechcore.scomponents.scell</groupId>
            <artifactId>api-impl</artifactId>
            <version>0.1.1-RC1</version>
            <scope>runtime</scope>
        </dependency>
        ...
    </dependencies>

*runtime scope is important for api-impl dependency

Start the API-library initialization:

    public class App extends Application {
        private static CompletableFuture<Optional<IScellApiResolver>> apiResolverFuture = ScellApiProvider.getApiResolverAsync();

        public static void main(String[] args) {
            apiResolverFuture.thenApply(apiResolver -> {
                //...
            });
            
            // ...
        }
    }

The call ScellApiProvider.getApiResolverAsync() aims to return the IScellApiResolver implementation which will use to resolve the API entry points or register custom implementations of API interfaces.

After getting the resolver you can resolve the API entry points:


public class App extends Application {
    private static CompletableFuture<Optional<IScellApiResolver>> apiResolverFuture = ScellApiProvider.getApiResolverAsync();
    private Optional<IScellApiResolver> apiResolver;

    @Override
    public void start(Stage stage) {
        //...
        apiResolver.ifPresent(resolver -> {
            // Resolve APIs providers
            ISpreadsheetApiProvider coreProvider = this.resolver.resolve(ISpreadsheetApiProvider.class);
            IScellUiApiProvider<Node> uiFxApiProvider = this.resolver.resolve(IScellUiFxApiProvider.class);

            CompletableFuture<IWorkbook> coreApi = coreProvider.createNew(); // Core API entry point
            IScellUiApi<Node> fxApi = uiFxApiProvider.create(coreApi); // Java FX control API entry point
            //...
        });
    }

    @Override
    public void init() throws Exception {
        this.apiResolver = apiResolverFuture.join();
    }

    public static void main(String[] args) {
        LauncherImpl.launchApplication(App.class, Preloader.class, args);
    }
}

The API entry points are resolved through coreProvider and uiFxProvider.

SCell API main entry points are: IWorkbook - core API entry point, IScellUiApi<Node> - UI API entry point for Java FX.

To create the Java FX API entry point you need the implementation of core IWorkbook interface. The IWorkbook you can get from the ISpreadsheetApiProvider by creation a new spreadsheet document (call createNew()) or loading an existing xlsx-file (call load()). You can work with the IWorkbook without the UI API to change or convert xlsx-documents.

To know what methods of the API Interfaces are implemented, see the SCell Public API Implementation Release Notes
